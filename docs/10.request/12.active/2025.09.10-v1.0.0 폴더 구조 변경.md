# Objective
- uq-d2s의 원격 repo와 로컬 repo의 폴더 구조를 변경하려고 함.
- 변경 이유
	1. WordPress의 child-theme 개발 프로젝트를 할 때, WordPress child-theme 폴더 전체를 git repo로 관리하지만, child-theme 내부에 uq-d2s의 repo를 다시 두어야 함.
	2. 현재 uq-d2s의 폴더 구조는 child-theme의 루트부터 시작하고 있음.

# Instruction
- 아래의 변경하려는 폴더 구조가 가능한지만 분석해서 알려주면 됨.
- 실제 폴더 구조 변경은 하지 않음.

# 폴더 구조
## WordPress child-theme의 일반적인 폴더 구조
```
    └── wp-content/
        │
        └── themes/
            └── {child-theme-name}/
                ├── functions.php
                ├── style.css
                ├── front-page.php      # Hierarchy-based Templates
                │
                ├── templates/          # Custom Page Templates
                │  	├── page-{page-name}.php
                │   └── page-about-us.php
                │
                ├── assets/
                │  	├── images/
                │	│	├── sections/
                │   │	└── logo/
                │   ├── css/
                │	│	├── front-page.css
                │   │	└── about-us.css
                │   └── js/
                │		├── front-page.js
                │		└── about-us.js
                │       
                └── includes/
                    ├── gp-customize.php
                    └── gp-blog.php
                
```

## 바꾸려고 하는 구조.

- 핵심은 d2s-v1으로 만든 page template .php 파일을 child-theme 바로 하위의 `./templates` 폴더가 아니라 `./d2s-v1/page-templates`로 옮길 수 있는지에 대한 여부이다.
- 일반적인 child-theme의 페이지 템플리트는 `./templates` 폴더에 두고, d2s-v1를 통해 만든 페이지 템플리트만 `./d2s-v1/page-templates`로 옮길 수 있는지에 대한 여부 확인이 필요한다.
